**O que cada pasta do Flutter em VS Code faz:**

**.dart_tool:** Guarda dados temporários e ferramentas que o Flutter precisa para compilar e rodar o seu app.

- Cache de pacotes (pub/): Informações sobre as dependências (as bibliotecas que você adiciona no pubspec.yaml) e como elas se relacionam.

- Código gerado automaticamente: Se você usa pacotes que geram código (como build_runner, freezed, json_serializable ou mobx), os arquivos temporários e mapeamentos da geração ficam guardados ali.

- Configurações de compilação: Arquivos que ajudam o Flutter a saber o que mudou no código para fazer o Hot Reload e o Hot Restart funcionarem de forma tão rápida.

---

**.idea:** Armazena as configurações do seu ambiente de trabalho local específicas para aquele projeto.

- Estilos de formatação de código que você configurou.

- Histórico de arquivos editados e posicionamento de janelas.

- Configurações de execução e depuração (Run/Debug configurations) que você criou na IDE.

- Plugins específicos do projeto que foram ativados.

---

**android:** A estrutura padrão de qualquer aplicativo Android nativo (geralmente escrito em Kotlin ou Java).

---

**ios:** A estrutura padrão de um aplicativo nativo para iOS (geralmente estruturado para usar Swift ou Objective-C).

---

**lib:** Onde fica todo o código-fonte em Dart do seu aplicativo. É onde você cria as telas, a lógica de negócios, as conexões com APIs, as animações e os componentes visuais.

---

**linux:** Contém um projeto nativo completo para o sistema operacional Linux.

---

**macos:** Contém um projeto nativo completo da Apple para o sistema operacional macOS.

---

**test:** É o espaço reservado no Flutter para você garantir que o seu aplicativo funciona corretamente antes de enviá-lo para os usuários. É nela que você escreve e armazena todos os seus testes automatizados.

---

**web:** Contém os arquivos de configuração e suporte necessários para compilar e rodar o seu aplicativo Flutter diretamente nos navegadores de internet (como Google Chrome, Safari, Firefox e Edge).

---

**windows:** Contém um projeto nativo completo em C++ para o sistema operacional Windows.

---

**.gitignore:** Não é exclusivo do Flutter, mas sim uma ferramenta fundamental do Git (o sistema de controle de versão que você usa para enviar seu código para o GitHub, GitLab, etc.).

---

**.metadata:** É um arquivo de controle interno oculto criado e utilizado automaticamente pelas ferramentas do Flutter SDK.

---

**analysis_options.yaml:** É o "manual de boas maneiras" do seu projeto Flutter. Ele serve para configurar o Linter e o Analisador estático do Dart.

---

**pubspec.lock:** Enquanto no pubspec.yaml você diz quais dependências você gostaria de usar, o pubspec.lock registra a versão exata e definitiva de cada pacote (e das subdependências deles) que foi realmente instalada no seu projeto.

---

**pubspec.yaml:** É o arquivo de configuração mais importante do seu projeto Flutter. Ele funciona como a certidão de nascimento e o manifesto central do seu aplicativo.
