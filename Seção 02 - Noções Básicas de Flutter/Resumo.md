Criamos uma classe que extende um componente sem estado (StatelessWidget)

No método main, podemos colocar ao invés de abertura de chaves ({}), podemos colocar como uma função Arrow (=>)

**@override** - Obrigatoriamente o componente StatelessWidget precisa implementar.

(Sub escrevendo o método build, este método irá ser chamado pelo próprio Flutter)

Cada componente tem o seu próprio contexto.

Cada componente **filho** se comunica com o componente **pai**.
